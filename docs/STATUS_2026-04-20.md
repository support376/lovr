## 오늘 작업한 것

### 1. 관계 화면 액션-우선 재구성 (완료)
**문제**: 상단에 스타일 선택 카드 + 목표 카드가 쌓여 있어서, 정작 "내가 할 행동"이 최하단에 묻힘. 유저가 설정 도구처럼 느낌.

**해결**:
- `src/app/r/[id]/page.tsx` 전면 개편
- **최상단**: `QuickActionCTA.tsx` (신규) — 큰 primary 버튼 `지금 행동 받기` / `다시 업데이트`
  - 목표 없으면 카테고리 pill 6개 인라인 → 원클릭으로 목표 저장 + 전략 제안 동시
- **중간**: `StrategyCards` — A/B/C 전략
- **하단 접힘**: `<details>` 로 목표/스타일/description 요약. 펼치면 StylePicker + 목표 변경 링크

### 2. 전략 포맷 3필드 명령조로 압축 (완료)
**문제**: 구포맷이 6필드(근거/절차/타이밍/성공 신호/실패 신호/리스크)라 장황. UI는 그중 3개만 쓰고 나머진 생성하고 버림. 제목도 "전략 A: 회피형 접근" 식 분석조라 "OO해라" 명령감이 안 남.

**해결**:
- `src/lib/prompts/loader.ts` `STRATEGY_PROPOSAL` 재작성
  - 출력: `## 지금 상황` + `### 1./2./3. [명령조 제목]` + `근거` / `왜` / 선택적 메시지 초안·타이밍
  - 제목 톤 강제: "~해라" / "~하지마" / "~먼저"
- `src/lib/strategies/parse.ts` — 신포맷 + 구포맷 둘 다 호환 (기존 Action 레코드 여전히 표시됨)
- `src/app/r/[id]/StrategyCards.tsx` — 번호 배지(1/2/3) + 볼드 명령 제목 → `근거` / `왜` 2블록 + 메시지 초안 박스

## 남은 큰 이슈

### ★ 1. 로그인 / 멀티유저 (내일 최우선)

**현재 상태**:
- 로그인 기능 없음
- 전체 앱이 `SELF_ID = 'actor-self'` 하드코딩 (`src/lib/actions/self.ts:9`)
- 1개 DB에 1명 분의 데이터만 사는 구조

**필요 작업 — 2 레이어**:

**A. 인증** (추천: Auth.js v5)
- Google OAuth + Kakao OAuth (한국 유저 카카오 필수)
- 이메일 매직링크는 옵션
- Vercel + Next.js 15 App Router 공식 지원

**B. 데이터 격리 — 두 길**:

| 방식 | 장점 | 단점 |
|------|------|------|
| 모든 테이블에 `user_id` 컬럼 | 전통적 SaaS, 운영 익숙 | 모든 쿼리 수정, 실수 1번 = 타인 데이터 누출 |
| **Turso per-user DB** | 완전 격리, 코드 거의 그대로 | DB 인스턴스 늘어남, schema 배포 일일이 |

**추천 방향**: Auth.js + Turso per-user DB. 연애 사적 데이터는 격리 > 편의.

**순서**:
1. Auth.js 붙이고 Google/Kakao 로그인 작동 확인
2. 공용 DB에 `users` 테이블 1개 추가 (userId → tursoDbUrl 매핑)
3. `db/client.ts` 수정해서 세션 기준으로 connection 동적 교체
4. 기존 로컬 데이터는 운영자(support@dreamframe.org) 계정으로 이전

### 2. 전략 생성 속도 (퀵 픽스 대기)

**현재 느린 원인** (유저 관측: 오래 걸림):
- `proposeStrategy` (`src/lib/engine/coach.ts`) input 구성:
  - system 프롬프트 (~2KB, 캐시됨)
  - ctx.markdown — 최근 이벤트 **30개 전문** (수 KB)
  - ontology block (stages/goals/styles 정의)
  - **plays 카탈로그 블록 — 매칭된 play 전부 (각 15줄, 최대 10개)** ← 가장 큼
- `max_tokens = 3000` — 새 포맷은 1000 이내면 끝남

**적용 대기 중인 퀵 픽스**:
- `max_tokens` 3000 → 1500
- plays block 텍스트 신포맷(`전략 A/B/C` → `추천 행동 1/2/3`) 맞춤 — 현재 프롬프트 지시가 충돌
- plays 개수 top 4 cap
- 이벤트 30 → 15
- user msg에 ontology/plays 캐시 추가
- (별도) 스트리밍 응답

내일 auth 전에 먼저 처리하든, 함께 처리하든 유저 판단.
